# Тест-кейси для сайту GreenCity

## TC-1: Реєстрація нового користувача (Sign Up)
**Preconditions:** Користувач не авторизований, відкрита головна сторінка сайту.

| Step | Action | Data | Expected Result |
|:---:|---|---|---|
| 1 | Натиснути кнопку "Sign Up" | - | Відкривається форма реєстрації |
| 2 | Заповнити всі обов'язкові поля валідними даними | Email, UserName, Password | Дані прийняті системою |
| 3 | Натиснути кнопку "Sign Up" | - | Акаунт створено успішно |

## TC-2: Фільтрація подій за статусом "Upcoming"
**Preconditions:** Відкрита сторінка https://www.greencity.cx.ua/#/greenCity/events

| Step | Action | Data | Expected Result |
|:---:|---|---|---|
| 1 | Натиснути на фільтр "Event time" та обрати "Upcoming" | Upcoming | Список оновлюється |
| 2 | Перевірити відображені події | - | На сторінці відображаються лише майбутні події |

## TC-3: Авторизація користувача (Sign In)
**Preconditions:** Користувач має зареєстрований акаунт.

| Step | Action | Data | Expected Result |
|:---:|---|---|---|
| 1 | Натиснути кнопку "Sign In" | - | Відкривається вікно входу |
| 2 | Ввести вірний Email та Password | Валідні дані | Дані введено без помилок |
| 3 | Натиснути кнопку "Sign In" | - | Користувач успішно заходить в акаунт (Expected Result) |

> **Note:** Під час виконання тесту було виявлено дефект. Попри введення коректних даних (Email та Password), які були збережені у системі, авторизація не відбулася. З'являється повідомлення "Bad email or password". Це свідчить про ймовірну помилку на стороні сервера або бази даних сайту.
